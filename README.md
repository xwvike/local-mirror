# Local Mirror

局域网目录镜像同步工具。把一台机器（服务端，reality）上的目录**单向**镜像到另一台机器（客户端，mirror），基于自定义 TCP 二进制协议，无外部服务依赖。

```
┌──────────────┐    自定义 TCP 协议     ┌──────────────┐
│   reality    │ ────────────────────▶ │    mirror    │
│  （服务端）    │   目录树 / 变更 / 文件  │  （客户端）    │
└──────────────┘                       └──────────────┘
      监听文件变化                          持续拉取，保持一致
```

> ⚠️ 镜像是单向的：客户端本地多出来的文件会被**删除**，以保持与服务端完全一致。

## 特性

- **三层同步机制**
  - 文件系统事件监听（fsnotify）：服务端实时记录变更目录
  - 增量追踪：客户端按时间窗游标周期性拉取"最近变更的目录"，只比对最小路径集合
  - 全量扫描：客户端周期性递归比对整棵目录树，作为最终一致性的安全网
- **内容级比对**：基于 blake3 哈希 + 文件大小判断差异，不会重复传输未变化的文件
- **可靠传输**：分块传输，落盘走"临时文件 → 全文件哈希校验 → 原子改名"，不会留下半截文件
- **热度评分监视**（应对 fsnotify 的 watch 数量上限）：目录按事件频率/路径特征打分，
  热门目录进 tier1 实时监听，冷门目录进 tier2 低频轮询
- **端口自动探测**：服务端从 52345 起自动选择可用端口；客户端在同一范围内握手探测，
  不会误连到恰好占用端口的其他程序
- **断线自愈**：客户端自动重连并通过 Reverify 确认仍是同一台服务端，失败则整体重建会话

## 快速开始

```bash
# 服务端：cd 到要共享的目录
cd /path/to/source && local-mirror -m reality

# 客户端：cd 到接收镜像的目录
cd /path/to/replica && local-mirror -m mirror -r 192.168.1.100
```

启动后会输出状态横幅（实际监听端口、同步目录、日志位置等）：

```
────────────────────────────────────────────────
  Local Mirror v1.0.0  ·  reality (服务器)
────────────────────────────────────────────────
  同步目录  /path/to/source
  监听地址  0.0.0.0:52345
  实例 ID   3b7b81ee
  进程 PID  62289
  日志      .local-mirror/logs/error.log (级别 error)
────────────────────────────────────────────────
```

## 参数

| 参数 | 说明 | 默认值 |
|---|---|---|
| `-m, --mode` | 运行模式：`reality`（服务端）/ `mirror`（客户端） | `reality` |
| `-r, --realityip` | 服务器 IP，仅客户端；空值回退连接 127.0.0.1 | 空 |
| `-c, --cooldown` | 全量扫描间隔（秒），仅客户端 | `300` |
| `-d, --diffinterval` | 增量变更查询间隔（秒），仅客户端 | `10` |
| `-f, --filebuffersize` | 文件传输分块大小（字节），仅服务端 | `65536` |
| `-l, --loglevel` | 日志级别：`debug` / `info` / `warn` / `error` | `error` |
| `-h, --help` | 显示帮助 | |
| `-v, --version` | 显示版本 | |

完整说明见 `local-mirror --help`。

## 构建

```bash
# 本机构建
go build -o local-mirror ./cmd/local-mirror

# 交叉编译全平台（输出到 dist/，版本号自动取 git describe）
./build.sh        # Linux / macOS
./build.ps1       # Windows
```

## 工作原理

1. **启动建树**：两端各自递归扫描同步目录，并发计算文件 blake3 哈希，
   目录树持久化到 bbolt（`.local-mirror/cache.db`，每次启动重建）。
2. **服务端监听**：fsnotify 事件（含新目录的递归补扫）实时更新目录树，
   变更目录按落库时间记录在一小时滚动窗口内，供客户端按时间段查询。
3. **客户端同步**：
   - 每 `diffinterval` 秒查询 `[上次游标, 现在]` 内的变更目录，仅比对这些目录及其新增子树；
   - 每 `cooldown` 秒全量递归比对整棵树；
   - 差异按 创建目录 / 下载文件 / 删除多余项 逐条执行，下载经哈希校验后原子落盘。

## 目录结构

```
cmd/local-mirror/    入口：参数校验、启动横幅、生命周期
config/              flag 定义与全局配置
internal/
  network/           自定义 TCP 协议（编解码、服务端、客户端、端口探测）
  tree/              目录树构建与 bbolt 持久化、变更目录记录
  watcher/           热度评分监视器与事件过滤
  *.go               reality / mirror 两种模式的编排逻辑
pkg/
  stack/             泛型栈
  utils/             哈希、路径规范化、忽略规则等工具
docs/                开发笔记
```

## 运行时产物

工作目录下会生成 `.local-mirror/`（已被同步逻辑和 git 忽略）：

- `cache.db` — 目录树缓存
- `logs/error.log` — 运行日志（错误同时输出到终端）
